# PRD: 圖書館理系統

## 1. 專案概述
- **背景與動機**：圖書館每天面臨大量的書籍進出、借閱與歸還操作，需要一個數位化系統來提高管理效率，降低人工記錄的繁瑣與錯誤，並透過數據分析（如熱門借閱排行）提供更好的服務體驗。
- **目標用戶**：圖書館管理人員。
- **核心價值主張**：提供一個操作直覺、易於維護的圖書館理系統，讓管理員能高效管理書籍庫存、會員資訊及精準追蹤借還書紀錄。

## 2. 功能需求
1. **書籍管理與查找**
   - 作為 **管理人員**，我希望 **能新增、編輯與查找書籍（可透過書名、作者、ISBN、分類、庫存數量等欄位搜尋）**，以便 **隨時掌握館藏狀態並快速幫助讀者找書**。
2. **會員資料系統**
   - 作為 **管理人員**，我希望 **能建檔並管理會員的個人資料（包含姓名、電子郵件、電話、註冊日期）**，以便 **有效掌握讀者聯絡資訊並維持最新的會員名冊**。
3. **借閱與歸還記錄追蹤**
   - 作為 **管理人員**，我希望 **能登記每筆借閱記錄（包含借閱日期、應歸還日期、實際歸還狀態）**，以便 **追蹤未歸還的書籍並確保館藏不致流失**。
4. **複雜借閱關聯管理**
   - 作為 **系統本身（架構需求）**，我希望 **能支援一位會員借閱多本書，一本書擁有多筆不同借閱歷史的關聯結構**，以便 **完整且真實地保留讀者借閱行為的軌跡**。
5. **書籍熱門排行統計**
   - 作為 **管理人員**，我希望 **系統能自動統計並列出各類書籍的出借次數排行榜**，以便 **將受歡迎的熱門書籍推薦給客戶，提升館內書籍流通率**。

## 3. 非功能需求
- **技術限制**：後端基於 Flask 框架開發，前端套用 Jinja2 模板引擎進行渲染，並且使用 SQLite 作為關聯式資料庫系統。
- **效能考量**：資料庫設計須確保一對多與多對多關聯查詢（如：計算借閱次數與熱門排行）時具備良好的系統響應速度。
- **安全考量**：系統僅供內部管理員存取，未來需考慮引入身分驗證（Authentication）機制避免資料遭竄改，並加入防禦 SQL 注入攻擊（SQL Injection）等防護措施。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 基礎的書籍資料建檔與查詢功能（CRUD）。
  - 會員資料管理與維護。
  - 建立讀者與書籍之間的借閱記錄，能查看與變更歸還狀態。
- **Should Have (應該有)**：
  - 進階的書籍複合查詢功能（可混合作者、書名、分類縮小範圍）。
  - 熱門書籍的簡易排行榜（依出借次數排序）。
- **Nice to Have (有最好)**：
  - 對逾期未還書籍提供醒目提示或篩選清單。
  - 圖表化顯示目前的館藏與出借比例儀表板。
